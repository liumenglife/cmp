# 内网穿透与去云化控制面部署手册 (v1.0)

本手册旨在指导如何将 **Cloud Control Plane** 从云端迁移至本地 PC，并确保远程（如微信小程序）能够通过安全隧道稳定访问，同时解决与本地代理工具（Proxifier）的冲突。

---

## 1. 核心架构图 (去云化后)


- **移动端**：微信小程序通过 HTTPS 请求域名。
- **中转层**：Cloudflare Edge 接收请求并通过加密隧道传回本地。
- **本地层**：`cloudflared` 接收流量 -> `Caddy` 分发 -> `Control Plane` 响应。
- **隔离层**：Proxifier 确保 `cloudflared` 直连，而 `Claude Code` 走代理。

---

## 2. 方案选择：Cloudflare Tunnel (推荐)

对于微信小程序，**Cloudflare Tunnel** 是最优解，因为它免费提供合法的 HTTPS 证书和域名访问。

### 部署步骤：
1. **安装 cloudflared**：在 PC 上下载并安装 Cloudflare Tunnel 客户端。
2. **创建隧道**：
   - 运行 `cloudflared tunnel login` 进行授权。
   - 运行 `cloudflared tunnel create my-bridge` 创建隧道。
3. **配置路由**：
   - 将你的域名（如 `api.yourdomain.com`）在 CF 后台指向该隧道。
4. **本地转发**：
   - 在配置文件中将流量指向本地 `Cloud Control Plane` 端口（如 `http://localhost:8080`）。

---

## 3. 解决 Proxifier 冲突 (关键)

为了防止 Proxifier 拦截隧道流量导致连接断开，必须设置 **Direct (直连)** 规则。

### Proxifier 配置步骤：
1. 打开 **Profile** -> **Proxification Rules**。
2. 点击 **Add** 添加新规则：
   - **Name**: `Tunnel_Bypass`
   - **Applications**: `cloudflared.exe` (如果是 frp 则填 `frpc.exe`)
   - **Target Hosts**: Any
   - **Action**: **Direct**
3. **优先级**：确保此规则位于所有代理规则（Proxy/SOCKS5）之**上**。

---

## 4. Caddy 的进阶配置

在本地使用 Caddy 可以提供双重保护和更灵活的路由。

**Caddyfile 示例：**
```text
:8080 {
    reverse_proxy localhost:9000 { 
        header_up Host {host}
        header_up X-Real-IP {remote_host}
    }
}
